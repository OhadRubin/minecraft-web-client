# MVP Philosophy: Visual SKETCHPAD 3D Project

## Core MVP Principle

**Goal**: Prove that 3D spatial reasoning skills learned in Minecraft transfer to web agents and other domains.

**MVP Approach**: Build the minimum necessary to collect 50 manual trajectories and test the core hypothesis.

## What MVP Means Here

### ✅ Essential (In Scope)
- 3D annotation tools that work (basic colored markers)
- Data collection pipeline that captures spatial reasoning
- 50 manual trajectory examples
- Proof that training data enables transfer learning

### ❌ Nice-to-Have (Out of Scope)
- Perfect graphics or fancy UI
- Advanced computer vision for object detection
- Multiple game environments
- Complex data validation pipelines
- Sophisticated analytics dashboards
- Advanced training techniques (LoRA, quantization)
- Comprehensive benchmarking across 10+ domains

## MVP Decision Framework

**The Rule**: "Does this directly enable collecting the first 50 trajectories?"

- **If YES** → Essential, implement now
- **If NO** → Scope creep, defer until after Phase 1

## Why MVP Matters

1. **Speed**: Prove core hypothesis in weeks, not months
2. **Risk Management**: Validate feasibility before investing in polish
3. **Research Focus**: Spatial reasoning transfer is the novel contribution
4. **Iterative Learning**: Build, test, learn, repeat

## Current Status

**✅ MVP ACHIEVED**: Pygame data collection operational, 3D tools working, ready to collect 50 trajectories.

**Next**: Execute data collection, not feature development.
